Documentação como Pilar do Produto

Reposicionamento da funcionalidade de Documentação (RAG v0) de feature secundária para módulo de primeira classe. Análise, decisões e implementação.

Contexto e Problema

O Synvera possui dois módulos funcionalmente distintos: Procedimentos / Cálculo e Documentação oficial. A documentação — que abrange CBHPM 2022, CBHPM 2025-2026, Manual SBN Neurocirurgia 2018 e Manual de Cirurgia de Coluna 3ª ed. — estava estruturada como uma funcionalidade secundária escondida dentro da Procedure Page.

SintomaEvidênciaImpacto
Accordion colapsado por padrão DocumentSearchPanel — requer interação para revelar Funcionalidade invisível sem ação do usuário
Posicionado abaixo do fold page.tsx:300-308 — fora do grid principal, após o footer Usuário raramente chega até ele durante o fluxo normal
Separado do contexto de uso Dúvidas surgem dentro do workspace-panel; documentação estava fora Fricção desnecessária no momento de necessidade real
Única via de descoberta: navbar Link "Documentação" só no topo — invisível durante o workflow Produto não comunica que a documentação é um pilar
Objetivo: O produto deve comunicar implicitamente que existem dois grandes módulos — Procedimentos e Documentação — sem aumentar a complexidade visual ou criar elementos de navegação desnecessários.

Localização da Entrada Contextual — Justificativa UX

O DocBridge foi posicionado dentro do workspace-panel, imediatamente após o bloco de conteúdo clínico (ClinicalAdjustmentsPanel) e antes do fechamento do painel:

// workspace-panel (card-plush, p-8)
ProcedureSearchPanel      // entrada: qual procedimento?
SelectedCodesPanel        // dúvida: quais códigos selecionar?
SpineVariablesPanel       // dúvida: quantos níveis?
TeamFeesPanel             // dúvida: regra de auxiliares? anestesia?
ClinicalAdjustmentsPanel  // dúvida: urgência? pediatria?
[ empty state se sem procedimento ]
DocBridge                 // ← AQUI — entrada contextual para docs

Por que aqui? As dúvidas sobre regras — auxiliares, urgência, via de acesso, pediatria, múltiplos procedimentos — surgem exatamente enquanto o médico ajusta esses campos. O DocBridge está fisicamente no mesmo espaço onde a dúvida nasce, eliminando a fricção de navegar para outra área da tela.

Por que sempre visível? O DocBridge aparece em ambos os estados (com e sem procedimento). Quando há procedimento selecionado, o contexto é pré-preenchido na URL. Quando não há, a entrada ainda está disponível para consulta autônoma.

Alternativa consideradaDescartada por
Accordion abaixo do grid (comportamento anterior) Abaixo do fold; oculto por padrão; separado do contexto
Banner entre hero e grid Competiria com hierarquia visual; visível mas não contextual
Link apenas na navbar Não contextual; dependente de o usuário lembrar que existe
Sidebar ou menu adicional Aumenta complexidade visual; contraria o design system

DocBridge — Implementação
// components/procedure/DocBridge.tsx
import Link from "next/link";
import { BookOpen } from "lucide-react";

interface DocBridgeProps {
  contextQuery?: string;
}

export function DocBridge({ contextQuery }: DocBridgeProps) {
  const q = contextQuery?.trim();
  const href = q
    ? `/consulta-documental?q=${encodeURIComponent(q)}`
    : "/consulta-documental";
  const ctaLabel = q
    ? q.length > 26 ? q.slice(0, 24) + "…" : q
    : "Abrir documentação";

  return (
    <div className="mt-6 flex items-center justify-between gap-4
      rounded-2xl border border-slate-100 dark:border-slate-800
      bg-slate-50/50 dark:bg-slate-900/20 px-4 py-3.5">
      <div className="flex items-center gap-3">
        <div className="flex h-8 w-8 items-center justify-center
          rounded-xl bg-primary/8 dark:bg-primary/15">
          <BookOpen size={14} className="text-primary dark:text-blue-300" />
        </div>
        <div>
          <p>Dúvida sobre uma regra?</p>
          <p>CBHPM · Manual SBN · Manual de Coluna</p>
        </div>
      </div>
      <Link href={href}>{ctaLabel} →</Link>
    </div>
  );
}
